# Handoff notes (from Claude session, 2026-07-05)

## What just happened

Codex's original build was a hackathon demo: UI called a client-side mock
analyzer behind a fake delay, nothing persisted, map/dashboard rendered
hardcoded `lib/mockIssues.ts`, and the "Live civic intelligence" scores were
made-up formulas. This session rebuilt it into a working product. Verified
end-to-end in a browser: submit → persists → clusters → shows in map/feed/
dashboard → shareable public receipt.

## Current architecture

- **DB**: `lib/db.ts` — SQLite via Node's built-in `node:sqlite` (Node 22.13+,
  no native deps, no better-sqlite3 — that needed a native build and was
  intentionally avoided). File at `data/parareport.db`, auto-migrated +
  seeded from `lib/seedIssues.ts` on first run (seeds run through the real
  analyzer, so stored receipts always match what the UI would generate).
- **Repository**: `lib/store.ts` — CRUD, location-token clustering
  (`findClusterMatch`), `getClusterSummary`, `getDashboardStats`. All
  dashboard/map numbers are derived from this, nothing fabricated.
- **Analysis**: `lib/analyzer.ts` (deterministic rules engine, unchanged
  logic from original) + `lib/gemma.ts` (calls Gemma vision API if
  `GEMMA_API_KEY` env var set, falls back to rules per-field on bad/missing
  JSON). Every issue records `analysisEngine: "gemma" | "rules"`.
- **API**: `app/api/reports/route.ts` (POST create, GET list),
  `app/api/reports/[id]/route.ts` (GET one),
  `.../confirm/route.ts` (still-unresolved bump),
  `.../resolve/route.ts` (mark fixed, optional proof photo upload).
- **Uploads**: `lib/uploads.ts` — validates type/size, writes to
  `public/uploads/` (gitignored), returns base64 for Gemma vision calls too.
- **Pages**: `/` (report form → receipt), `/issues` (feed + filters + Leaflet
  map, `components/IssueMap.tsx`), `/issues/[id]` (public shareable receipt,
  OG metadata, confirm/resolve actions via `components/IssueActions.tsx`),
  `/dashboard` (real stats).
- **Removed**: `lib/mockIssues.ts`, `lib/intelligence.ts`, `lib/clustering.ts`,
  `components/AppShell.tsx`, `CommandCenter.tsx`, `IntelligencePanel.tsx`,
  `ParaMap.tsx`, `WardDashboard.tsx`, `SeasonalDashboard.tsx`,
  `IssueClusterCard.tsx`, hero image in `public/visuals/`. Old
  `app/api/analyze/route.ts` replaced by `app/api/reports/*`.

## Known gaps / not yet done

1. **No tests** — nothing automated beyond the manual browser pass above.
   Consider adding at least a couple of API route tests (create → cluster →
   confirm → resolve) if continuing this work.
2. **Clustering is naive** — `findClusterMatch` in `lib/store.ts` does simple
   location-token overlap matching, scoped to same `mode`. No fuzzy/geo
   radius matching. Good enough for demo-quality dedup, not production.
3. **No auth** — all API routes are unauthenticated by design (MVP scope per
   original spec). If this becomes a real product, `/api/reports/[id]/resolve`
   and `/confirm` need some abuse protection (rate limit at minimum).
4. **Deploy target undecided** — SQLite + local file uploads means it needs a
   host with persistent disk (Fly.io/Railway/Render/VPS), not Vercel/serverless
   as-is. README has notes; actual deploy not attempted this session.
5. **Gemma path untested live** — no `GEMMA_API_KEY` was set during this
   session, so the Gemma branch in `lib/gemma.ts` only got a structural
   review, not a live API call. Worth testing with a real key before trusting
   it in front of anyone.
6. **Hindi share text** — still generated by a template string in
   `analyzer.ts` (`createHindiShareText`), not verified by a Hindi speaker.
   Same caveat likely applies to Bengali text — flagged in the pilot's other
   project CLAUDE.md as needing human verification, worth applying that same
   discipline here since register/politeness in Bengali civic language matters.
7. **Old dev server** was occupying port 3000 during this session (unrelated
   process, PID 47062) — preview ran on an auto-assigned port instead. Not
   this project's concern, just noting in case `npm run dev` seems to silently
   pick a different port for you too.
